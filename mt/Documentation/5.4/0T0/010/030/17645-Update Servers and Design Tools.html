Overview

Explains how to update the BA and DI Servers as well as the Design Tools.

Update Your BA Server

These instructions are for users who installed Pentaho 5.1.x, 5.2.x, or 5.3.x using the Wizard or their own repository.

  1. If you have not already done so, prepare your environment .
  2. At a command prompt, run the BAServerConfigAndSolutionsBackup  utility to create three .zip files in the current user’s home folder. The utility receives one parameter, the path to the existing tomcat and pentaho-solutions  folders.

Windows example: 

BAServerConfigAndSolutionsBackup "C:\Program Files\pentaho\server\biserver-ee"

Linux example: 

BAServerConfigAndSolutionsBackup '/opt/custom software/pentaho/server/biserver-ee'
  1. Move the existing pentaho/server/biserver-ee/tomcat/webapps/pentaho folder to a location outside the Pentaho folder structure.

Windows example: 

C:\Windows\temp or C:\temp

Linux example: 

/tmp
  1. From the unzipped biserver-ee-5.4 directory, copy the new tomcat/webapps/pentaho  folder to the existing pentaho/server/biserver-ee/tomcat/webapps/ directory.
  2. Move the pentaho/server/biserver-ee/pentaho-solutions/system folder to a location outside Pentaho folder structure.

Windows example :

C:\Windows\temp or C:\temp

Linux example :

/tmp
  1. From the unzipped biserver-ee-5.4 directory, copy the pentaho-solutions/system folder to the existing pentaho/server/biserver-ee/pentaho-solutions/ directory.
  2. Move the downloaded plugin files you are licensed to use to the pentaho/server/biserver-ee/pentaho-solutions/system folder.
  3. At a command prompt, run the BAServerConfigAndSolutionsRestore  utility to restore the three .zip files from the current user’s home folder. The utility receives one parameter, the path to the folder containing the tomcat and pentaho‐solutions folders.

Windows example: 

BAServerConfigAndSolutionsRestore "C:\Program Files\pentaho\server\biserver-ee"

Linux example: 

BAServerConfigAndSolutionsRestore /opt/home/pentaho/server/biserver-ee
  1. Restart the server. This link shows you how:  https://help.pentaho.com/Documentation/5.4/0H0/060/010/000

If you do not wish to upload or replace your current sample data, it is recommended that you rename the files located in the pentaho-solutions/system/default-content directory prior to restarting the server. These files may include pentaho-operations-mart-etl, pentaho-operations-mart-operations, plugin-samples-zip, and samples-zip .

Update Your DI Server

These instructions are for those users who installed Pentaho 5.1.x, 5.2.x, or 5.3.x using the Wizard or their own repository.

  1. If you have not done so already, prepare your environment .
  2. At a command prompt, run the DIServerConfigAndSolutionsBackup  utility to create three .zip files in the current user’s home folder. The utility receives one parameter, the path to the existing tomcat and pentaho-solutions  folders.

Windows example: 

DIServerConfigAndSolutionsBackup "C:\Program Files\pentaho\data-integration-server"

Linux example: 

DIServerConfigAndSolutionsBackup '/opt/custom software/pentaho/data-integration-server'
  1. Move the existing pentaho/server/data-integration-server/tomcat/webapps/pentaho-di  folder to a location outside the Pentaho folder structure.

Windows example: 

C:\Windows\temp or C:\temp

Linux example: 

/tmp
  1. From the unzipped pdi‐ee‐5.4 directory, copy the new tomcat/webapps/pentaho-di  folder to the existing pentaho/server/biserver-ee/tomcat/webapps/ directory.
  2. Move the folder pentaho/server/data-integration-server/pentaho-solutions/system to a safe location outside the pentaho folder structure.

Windows example: 

C:\Windows\temp or C:\temp

Linux example: 

/tmp
  1. From the unzipped data‐integration‐server 5.4 file, copy the pentaho-solutions/system directory to the existing pentaho/server/data-integration-server/pentaho-solutions/ directory.
  2. At a command prompt, run the DIServerConfigAndSolutionsRestore  utility to restore the three .zip files from the user’s home directory. The utility receives one parameter: the path to the folder containing the tomcat and pentaho‐solutions folders. 

Windows example: 

BAServerConfigAndSolutionsRestore "C:\Program Files\pentaho\data-integration-server"

Linux example: 

BAServerConfigAndSolutionsRestore /opt/home/pentaho/data-integration-server
  1. Restart the server. This link shows you how:  https://help.pentaho.com/Documentation/5.4/0F0/0O0/040

Update Your Design Tools

The design tools are stand-alone tools, and can be updated on workstations as follows.

  1. Stop the design tool if it’s running. These links show you how: Start and Stop BA Design Tools  or Start Spoon .
  2. Move the old design tool out of the Pentaho folder structure to a temporary folder.
  3. Copy the new version of the design tool into the Pentaho folder structure.
  4. Restart your design tool.
